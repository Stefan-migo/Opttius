# 📦 Archivo de Documentación

Documentación obsoleta, duplicada o histórica del proyecto.

## Contenido

Este directorio contiene:

1. **Documentos con fecha (2026-02, etc.):** Snapshots históricos
2. **Archivos de análisis obsoletos:** Análisis reemplazados por nueva documentación
3. **Documentación de "salvataje":** Recuerdos del proceso de rescate del código
4. **Carpetas legacy:** Estructuras de carpetas anteriores

## Cuándo Mover al Archivo

1. Cuando un documento ha sido **reemplazado** por uno más actual
2. Cuando el documento es un **snapshot** de un estado anterior
3. Cuando es un **análisis preliminar** ya superado
4. Cuando tiene una **fecha específica** en el nombre (ej: `_2026-02.md`)

## Cuándo NO Archivar

1. Documentos técnicos de referencia (esquemas de DB, APIs)
2. Guías de usuario aún válidas
3. Documentación de módulos activos
4. Documentación de arquitectura principal
